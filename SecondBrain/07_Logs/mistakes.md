# 실수 기록

## 목적
이 문서는 반복하지 않아야 할 실수와 개선점을 기록한다.

## 핵심 내용
- 지식 구조가 복잡해질수록 고립 문서와 중복 문서가 생길 수 있다.
- 자동화 전에는 안전 규칙과 백업 규칙을 먼저 확정해야 한다.
- 기능 욕심 때문에 Markdown 원본 구조가 흔들리지 않게 주의한다.

## 요약
실수 기록은 시스템이 성장하면서 품질을 잃지 않게 하는 학습 문서다.

## 세부 내용
### 주의할 실수
- 빈 문서 생성
- 내부 링크가 없는 고립 문서 생성
- 업데이트 로그 누락
- 원본 Markdown이 아닌 캐시를 원본처럼 취급
- 사용자 승인 없는 삭제 또는 덮어쓰기

실수가 발생하면 비난보다 원인, 영향, 재발 방지 규칙을 기록한다.
