# 대시보드 UI

## 목적
이 문서는 AI비서 대시보드와 AI 채팅 화면의 구체적인 UI 구성을 정리한다.

## 핵심 내용
- 전체 대시보드는 참고 이미지의 레이아웃을 기준으로 한다.
- AI 채팅 화면은 왼쪽 채팅창, 오른쪽 지식 네트워크 데이터 패널을 핵심으로 한다.
- 사이드바의 지식 그래프 메뉴는 제외한다.

## 요약
대시보드 UI는 대화와 지식 연결을 한 화면에서 확인하게 만드는 업무형 인터페이스다.

## 세부 내용
### 전체 대시보드
- 좌측: 주요 메뉴
- 상단: 검색/명령 입력, AI 상태, 새로 만들기
- 중앙: 현재 선택된 작업 영역
- 우측: AI 활동 요약, 최근 활동, 연결된 에이전트, 빠른 액션
- 하단: 오늘 일정, 우선순위 작업, AI 추천 인사이트, 저장소 현황

### AI 채팅 화면
- 왼쪽 패널: 사용자와 AI의 대화 흐름
- 오른쪽 패널: 대화에서 추출된 문서, 태그, 연결 후보, 업데이트 후보
- 오른쪽 패널의 항목: 관련 문서명, 연결 이유, 연결 강도, 추천 액션
- 채팅 입력창: 파일 참조, 내부 링크 삽입, 전송 버튼

### 제외
- CRM 화면
- 자동화 실행 화면
- 실제 그래프 엔진
- 검색/RAG/임베딩 기능

## AI 메모
AI는 오른쪽 지식 네트워크 패널을 실제 그래프 시각화가 아니라 대화에서 누적되는 맥락 목록으로 먼저 취급한다.

## AI가 알아야 할 규칙
화면 설계는 기능 구현 범위를 넘지 않게 작성하고, 구현 전 단계에서는 와이어프레임과 상태 정의에 집중한다.

## 다음 행동
- AI 채팅 화면의 빈 상태, 로딩 상태, 연결 추천 상태를 추가한다.
- 실제 컴포넌트 구현 단계에서 디자인 토큰을 분리한다.
