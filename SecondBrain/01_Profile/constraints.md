# 제약 조건

## 목적
이 문서는 프로젝트와 AI 작업에서 지켜야 할 제약 조건을 정의한다.

## 핵심 내용
- 사용자 데이터는 로컬에 남아야 한다.
- 이번 단계에서는 AI 채팅, 검색, 임베딩, 데이터베이스 기능을 만들지 않는다.
- CRM과 자동화는 후속 단계에서 다룬다.

## 요약
제약 조건은 범위 확장과 데이터 위험을 통제하는 기준이다.

## 세부 내용
- 서버 저장 금지: 원본 지식은 서비스 서버에 저장하지 않는다.
- 삭제 금지: 기존 사용자 파일은 삭제하지 않는다.
- 백업 우선: 같은 파일 수정 전 백업을 만든다.
- 기능 범위: 1단계는 폴더 구조와 문서 템플릿 완성까지만 포함한다.

## AI 메모
AI는 작업 범위를 벗어나는 요청이 섞여 있어도 현재 단계의 완료 기준을 분명히 구분한다.

## AI가 알아야 할 규칙
자동화, 외부 동기화, 원격 저장, 삭제성 변경은 사용자 승인 없이는 실행하지 않는다.
