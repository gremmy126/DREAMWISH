# 선호

## 목적
이 문서는 사용자의 작업 방식, UI 방향, AI 응답 방식에 대한 선호를 저장한다.

## 핵심 내용
- 사용자는 로컬 파일과 Markdown 중심 구조를 선호한다.
- 대시보드는 참고 이미지처럼 조용하고 밀도 있는 업무형 UI를 지향한다.
- AI 채팅은 왼쪽 대화, 오른쪽 지식 네트워크 축적 구조를 우선한다.

## 요약
선호 문서는 제품 경험과 AI 행동의 기본 톤을 결정한다.

## 세부 내용
- 데이터 저장: 로컬 PC 우선
- 문서 형식: 순수 Markdown
- 연결 방식: `[[문서명]]` 내부 링크
- UI 방향: 가벼운 배경, 명확한 패널, 정보 중심 카드
- 제외 항목: 현재 단계에서 CRM, 자동화, 실제 지식 그래프 기능은 제외

## AI 메모
AI는 사용자 선호가 명시된 경우 일반적인 제품 관행보다 이 문서의 선호를 우선한다.

## AI가 알아야 할 규칙
추론으로 선호를 확정하지 말고, 임시 추론은 별도 표시 후 사용자 확인을 받는다.
