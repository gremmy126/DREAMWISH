# 에이전트 규칙

## 목적
이 문서는 향후 프로젝트, 디자인, 사업, 개발별 AI 에이전트를 만들 때의 기준을 정의한다.

## 핵심 내용
- 현재 단계에서는 실제 에이전트를 구현하지 않는다.
- 에이전트는 각자 담당 문서 범위와 권한을 가져야 한다.
- 모든 에이전트는 공통적으로 Local First와 Markdown First 원칙을 따른다.

## 요약
에이전트 규칙은 역할별 AI가 문서와 작업을 다루는 방식의 기준이다.

## 세부 내용
### 후보 에이전트
- 프로젝트 에이전트: 목표, 작업, 일정, 결정 연결
- 디자인 에이전트: 디자인 시스템, UI, 브랜드, 참고자료 연결
- 사업 에이전트: 아이디어, 수익 모델, 마케팅, 경쟁사 연결
- 개발 에이전트: 아키텍처, API, 데이터, 작업 연결

## AI 메모
AI는 에이전트 역할을 나누더라도 동일한 Markdown 원본을 공유한다는 점을 유지한다.

## AI가 알아야 할 규칙
에이전트는 사용자 승인 없이 문서를 삭제하거나 외부로 전송하지 않는다.
