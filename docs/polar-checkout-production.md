# Polar Checkout production URL checklist

Production domain:

```text
https://dreamwish.co.kr
```

Checkout URLs generated by the server:

```text
success_url=https://dreamwish.co.kr/payment/success?checkout_id={CHECKOUT_ID}
return_url=https://dreamwish.co.kr/pricing?payment=cancelled
```

The app uses Polar's REST API directly, so the checkout payload uses the REST field names:

```json
{
  "products": ["POLAR_PRODUCT_ID"],
  "success_url": "https://dreamwish.co.kr/payment/success?checkout_id={CHECKOUT_ID}",
  "return_url": "https://dreamwish.co.kr/pricing?payment=cancelled"
}
```

Railway Production variables:

```env
NEXT_PUBLIC_APP_URL=https://dreamwish.co.kr
POLAR_SUCCESS_URL=https://dreamwish.co.kr/payment/success
POLAR_CANCEL_URL=https://dreamwish.co.kr/pricing?payment=cancelled
```

After changing Railway variables, redeploy the Production service so Next.js receives the updated public and server env values.

Polar Dashboard allow/return URLs:

```text
https://dreamwish.co.kr/payment/success
https://dreamwish.co.kr/pricing
```

If Polar does not allow query strings in the dashboard allowlist, register only the base paths above. The server still sends the `checkout_id={CHECKOUT_ID}` query placeholder in `success_url`.

Local development example:

```env
NEXT_PUBLIC_APP_URL=http://localhost:3000
POLAR_SUCCESS_URL=http://localhost:3000/payment/success
POLAR_CANCEL_URL=http://localhost:3000/pricing?payment=cancelled
```

This repository's default dev script currently serves on `127.0.0.1:3100`; when testing locally on port 3100, set the local values to `http://localhost:3100` or run the dev server on port 3000.
