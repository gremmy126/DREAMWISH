# 자동화 앱 카탈로그·실행 작업·탭 기능 확장 설계

## 1. 목적

기존 MAKE 스타일 자동화 스튜디오를 실제로 구성하고 실행할 수 있는 앱 자동화 환경으로 확장한다. 사용자는 실제 앱 로고를 보고 모듈을 선택하고, 모듈마다 제공되는 트리거와 실행 작업을 검색해 선택하며, 앱별로 정확한 인증값을 입력한다. 템플릿, 실행 내역, 연결 관리, 사용 가이드 탭은 각각 독립된 실제 화면으로 동작한다.

이 문서는 `2026-07-13-make-automation-studio-design.md`를 대체하지 않고 다음 범위를 구체화한다.

- 실제 앱·웹사이트 로고와 확장 앱 카탈로그
- 앱별 인증 방식과 정확한 입력 필드
- 모듈별 트리거·실행 작업 선택
- 기본 작업에 없는 API를 위한 사용자 지정 API 요청
- 템플릿·실행 내역·연결 관리·사용 가이드 탭
- 실제 실행, 오류 처리, 보안, 검증

## 2. 확정된 접근법

앱마다 로고, 인증 스키마, 트리거, 실행 작업, 설정 필드, 출력 스키마를 하나의 타입 안전한 앱 레지스트리로 관리한다. 모든 앱에 공통 Secret 한 칸을 제공하는 방식은 사용하지 않는다.

OAuth 앱은 일반 사용자가 Client Secret을 입력하지 않는다. DREAMWISH 운영자가 Client ID, Client Secret, Redirect URI를 서버 환경변수에 설정하고, 일반 사용자는 `계정 연결` 버튼으로 로그인과 동의만 진행한다. 운영자 설정이 없는 OAuth 앱은 연결된 것처럼 표시하지 않고 `관리자 OAuth 설정 필요` 상태를 표시한다.

사용자 소유 키를 지원하는 앱은 해당 앱에 필요한 필드만 보여준다. 모든 비밀값은 암호화해 저장하고, 저장 후에는 원문을 다시 반환하지 않는다.

## 3. UI 구조

기존 사이드바는 변경하지 않는다. 자동화 페이지 본문은 제공된 MAKE 스타일 시안을 기준으로 유지하고 상단 탭을 실제 화면 전환으로 구현한다.

### 3.1 시나리오 탭

- 왼쪽: 검색 가능한 앱·도구 모듈 카탈로그
- 중앙: `@xyflow/react` 기반 노드·연결 캔버스
- 오른쪽: 선택한 시나리오 또는 모듈 설정 패널
- 상단: 실시간 실행, 실행, 테스트, 저장, 추가 메뉴
- 하단: 추천 템플릿 카드

앱 모듈을 추가하면 다음 순서로 설정한다.

1. 연결 계정 또는 자격증명 선택
2. 트리거 또는 실행 작업 선택
3. 대상 리소스와 필수 입력값 지정
4. 이전 모듈 출력값을 입력값에 매핑
5. 재시도, 타임아웃, 실패 시 중단 여부 지정
6. 모듈 단독 테스트

작업이 선택되지 않은 앱 모듈은 저장할 수 있지만 시나리오 활성화와 실제 실행은 할 수 없다. 캔버스에는 앱 로고, 선택한 작업명, 연결 상태, 실행 상태를 표시한다.

### 3.2 템플릿 탭

- 이름·설명·앱·업무 분야 검색
- 카테고리 필터: 이메일, 영업, CRM, 프로젝트, 파일, 마케팅, 소셜, AI, 데이터
- 템플릿 카드에 실제 앱 로고 흐름과 필요한 연결 표시
- `사용하기`를 누르면 사용자의 새 비활성 시나리오 사본 생성
- 연결 또는 필수 입력이 누락된 노드를 강조 표시
- 템플릿 사용만으로 자동 활성화하거나 즉시 실행하지 않음

### 3.3 실행 내역 탭

- 시나리오, 상태, 시작 방식, 날짜로 필터
- 실행 ID, 시나리오명, 시작 방식, 상태, 시작·종료 시각, 소요 시간 표시
- 실행 상세에서 모듈별 순서, 시도 횟수, 입력·출력 요약, 오류 코드·메시지 표시
- 실패한 실행 전체 재실행과 실패한 모듈부터 재실행 지원
- 키, 토큰, Authorization 헤더, 쿠키, 비밀번호는 로그와 미리보기에서 제거

### 3.4 연결 관리 탭

- 앱 로고, 앱 이름, 인증 방식, 연결 상태, 연결 계정, 마지막 검증 시각 표시
- 상태: 연결됨, 만료됨, 검증 실패, 관리자 설정 필요, 연결 안 됨
- 앱마다 정확한 입력 필드와 발급 위치 도움말 제공
- 저장 전 형식 검사, 저장 직후 실제 연결 검사
- 수정은 새 값으로 덮어쓰며 기존 원문 조회는 지원하지 않음
- 연결 해제와 삭제 전에 영향을 받는 시나리오 수 표시

### 3.5 사용 가이드 탭

- 계정 연결하기
- 시나리오 만들기
- 모듈 작업과 데이터 매핑하기
- 테스트·활성화·실행하기
- 실행 오류 해결하기
- 앱별 키 발급 위치와 입력 필드 표
- OAuth, API Key, Access Token, Bot Token, Application Password 용어 설명

상단 `가이드 보기` 버튼은 사용 가이드 탭으로 이동한다.

## 4. 컴포넌트와 모듈 경계

현재 큰 `AutomationView`는 다음 책임으로 분리한다.

- `AutomationStudio`: 페이지 상태와 탭 전환
- `AutomationAppLogo`: 로컬 SVG 브랜드 로고와 내부 도구 아이콘 렌더링
- `ModuleCatalog`: 앱·도구 검색과 필터
- `ScenarioCanvas`: 노드·엣지 편집
- `AutomationNode`: 로고, 작업명, 연결·실행 상태 표시
- `ModuleInspector`: 연결, 작업 선택, 입력 매핑, 오류 정책 편집
- `ActionPicker`: 트리거·조회·생성·수정·삭제·전송·고급 작업 검색
- `TemplateGallery`: 템플릿 탐색과 복사
- `RunHistory`: 실행 목록과 필터
- `RunDetail`: 모듈별 실행 상세
- `ConnectionManager`: 앱별 자격증명 추가·검증·삭제
- `AutomationGuide`: 사용법과 앱별 인증 안내

데이터 정의는 UI에서 분리한다.

- `AutomationAppDefinition`: 앱 ID, 이름, 로고, 브랜드 색상, 인증 스키마
- `AutomationActionDefinition`: 작업 ID, 종류, 리소스, 필수 입력, 출력, 실행기
- `AutomationCredentialSchema`: 사용자 필드, 운영자 필드, 비밀 여부, 검증 규칙
- `AutomationTemplateDefinition`: 카테고리, 노드, 엣지, 필수 연결

## 5. 로고 정책

- 브랜드 앱은 공식 브랜드 자산 또는 Simple Icons의 로컬 SVG를 저장해 사용한다.
- 외부 로고 CDN에 런타임 의존하지 않는다.
- Schedule, Router, Filter, Code, Delay, Iterator 같은 내부 모듈은 Lucide 또는 전용 내부 SVG를 사용한다.
- 글자 한 글자로 만든 임시 로고는 제거한다.
- 모든 로고는 고정 비율 컨테이너 안에서 잘리지 않고 텍스트를 밀어내지 않도록 한다.
- 다크·라이트 배경에서 식별되지 않는 단색 로고에는 브랜드 배경 또는 테두리를 적용한다.

## 6. 앱별 인증 필드

`사용자 입력`은 일반 사용자가 연결 관리 화면에서 입력하는 값이다. `운영자 설정`은 DREAMWISH 서버 환경변수에만 저장하는 값이다.

| 앱 | 인증 방식 | 사용자 입력 | 운영자 설정 |
|---|---|---|---|
| Gmail, Google Sheets, Google Calendar, Google Drive, YouTube | Google OAuth | 계정 연결 | Client ID, Client Secret, 정확한 Redirect URI |
| Slack | OAuth | 계정 연결 | Client ID, Client Secret, Signing Secret, Redirect URI |
| Notion | Internal Integration | Integration Token | 없음 |
| GitHub | Fine-grained PAT | Personal Access Token | 없음 |
| Discord | Bot | Bot Token, 서버·채널 ID | 없음 |
| Telegram | Bot | Bot Token, Chat ID | 없음 |
| Outlook, Microsoft Teams, OneDrive | Microsoft OAuth | 계정 연결 | Client ID, Client Secret, Tenant 설정, Redirect URI |
| Dropbox | OAuth | 계정 연결 | App Key, App Secret, Redirect URI |
| Airtable | PAT | Personal Access Token | 없음 |
| Trello | API Key + Token | API Key, API Token | 없음 |
| Asana | PAT | Personal Access Token | 없음 |
| Jira | API Token | 사이트 URL, 계정 이메일, API Token | 없음 |
| Linear | Personal API Key | Personal API Key | 없음 |
| HubSpot | Private App | Private App Access Token | 없음 |
| Salesforce | Token | Instance URL, Access Token | 없음 |
| Stripe | API Key | Restricted Key 권장 또는 Secret Key | 없음 |
| Shopify | Admin API | 스토어 도메인, Admin API Access Token | 없음 |
| WordPress | Application Password | 사이트 URL, 사용자명, Application Password | 없음 |
| Facebook | Meta Token | Page Access Token, Page ID | 없음 |
| Instagram | Meta Token | Access Token, Instagram Business Account ID | 없음 |
| X | OAuth 1.0a 사용자 자격증명 | API Key, API Secret, Access Token, Access Token Secret | 없음 |
| LinkedIn | OAuth Access Token | Access Token, 사용자 또는 조직 ID | 없음 |
| OpenAI | API Key | API Key | 없음 |
| Webhook | Webhook | Webhook URL, 선택 Signing Secret | 없음 |
| HTTP 요청 | 사용자 지정 | URL과 인증 유형에 따른 동적 필드 | 없음 |

HTTP 요청 모듈은 인증 유형을 `없음`, `API Key`, `Bearer Token`, `Basic Auth`, `OAuth Access Token` 중에서 선택한다. 선택에 따라 Header 이름, API Key, Token, 사용자명, 비밀번호 필드만 보여준다.

Schedule, CRM, AI 분석, Router, Filter, Code, Delay, Iterator에는 별도 외부 자격증명이 없다. `AI 분석`은 DREAMWISH 서버의 승인된 AI 공급자 설정을 사용하며, 사용자가 직접 연결하는 OpenAI 모듈과 구분한다.

## 7. 모듈별 실행 작업

작업 선택기는 다음 그룹을 제공한다.

- 트리거
- 조회·검색
- 생성·전송
- 수정
- 삭제·보관
- 고급

### 7.1 메시지와 이메일

- Gmail·Outlook: 새 이메일, 이메일 검색·조회, 전송, 답장, 임시저장, 라벨·폴더 이동, 첨부파일 조회
- Slack·Teams: 새 메시지, 메시지 전송·수정·삭제, 채널·사용자 조회, 반응 추가, 파일 업로드
- Discord: 메시지 전송·수정·삭제, 채널·멤버 조회, 반응·역할 작업
- Telegram: 새 업데이트, 메시지·사진·문서 전송, 메시지 수정·삭제, 채팅 정보 조회

### 7.2 파일과 데이터

- Google Drive·OneDrive·Dropbox: 새 파일, 검색·목록·다운로드, 업로드, 복사, 이동, 공유, 삭제
- Google Sheets: 행 감지, 범위·행 조회, 찾기, 행 추가·수정·삭제, 시트 생성
- Airtable: 레코드 감지, 목록·검색·조회, 생성·수정·삭제
- Notion: 데이터베이스 조회, 페이지·블록 조회, 페이지 생성·수정·보관, 블록 추가
- WordPress: 글·페이지·미디어·댓글 조회, 생성, 수정, 삭제

### 7.3 일정과 프로젝트

- Google Calendar: 일정 생성·변경 감지, 목록·조회, 생성, 수정, 참석자 추가, 취소·삭제
- Trello: 보드·리스트·카드 조회, 카드 생성·수정·이동·보관, 댓글 추가
- Asana: 프로젝트·섹션·작업 조회, 작업 생성·수정·완료, 댓글 추가
- Jira: 프로젝트·이슈 검색·조회, 이슈 생성·수정, 전환, 댓글 추가
- Linear: 프로젝트·사이클·이슈 검색·조회, 이슈 생성·수정, 상태 변경, 댓글 추가
- GitHub: 저장소·이슈·PR·커밋·릴리스 조회, 이슈·PR·댓글 생성, 워크플로 실행

### 7.4 CRM, 결제와 상거래

- 내부 CRM: 고객·회사·거래 검색, 생성, 수정, 메모·활동 추가
- HubSpot: 연락처·회사·거래·티켓 검색, 생성, 수정, 삭제
- Salesforce: 객체 설명, SOQL 조회, 레코드 생성·수정·Upsert·삭제
- Stripe: 고객·상품·가격·Payment Intent·구독·청구서·환불 조회·생성·수정과 이벤트 트리거
- Shopify: 상품·주문·고객·재고·배송·메타필드 조회·생성·수정

### 7.5 소셜과 AI

- Facebook: 페이지 게시물·사진·동영상 게시, 댓글 조회·답글, 게시물 삭제, 인사이트 조회
- Instagram: 이미지·캐러셀 게시, 댓글 조회·답글, 미디어·인사이트 조회
- X: 게시물 생성·삭제·검색, 사용자 조회
- LinkedIn: 사용자·조직 조회, 게시물 생성·삭제, 조직 통계 조회
- YouTube: 채널·영상·댓글·재생목록 조회, 댓글 작성·답글, 재생목록 항목 추가
- OpenAI: 텍스트·구조화 응답, 임베딩, 이미지 생성, 음성 인식, 음성 합성

### 7.6 내부 제어 모듈

- Schedule: 지정 시각, 간격, Cron 트리거
- Webhook: 서명 검증이 가능한 수신 트리거 또는 외부 Webhook 호출
- Router: 조건별 여러 경로 분기
- Filter: 조건이 참일 때만 다음 단계 실행
- Iterator: 배열 항목 반복
- Code: 제한된 입력·출력 계약 안에서 코드 변환
- Delay: 지정 시간 대기 또는 지정 시각까지 대기
- AI 분석: 요약, 분류, 정보 추출, 구조화, 승인된 이미지 생성

### 7.7 MAKE형 데이터·제어 도구

기존 Router, Filter, Code, Delay, Iterator 외에 다음 내부 도구를 실제 실행 가능한 모듈로 추가한다.

| 도구 | 선택 가능한 작업 |
|---|---|
| Text·Formatter | 텍스트 결합, 분리, 치환, 대소문자 변환, 공백 정리, 템플릿 렌더링, 정규식 추출 |
| 날짜·시간 | 파싱, 형식 변환, 시간대 변환, 날짜 더하기·빼기, 기간 계산, 현재 시각 생성 |
| Math | 사칙연산, 반올림, 최소·최대, 합계·평균, 백분율, 수식 계산 |
| JSON | 파싱, 직렬화, 경로 선택, 키 추가·삭제, 객체 병합, 스키마 검증 |
| CSV | CSV 파싱·생성, 열 선택·변환, 행 필터, 배열 변환 |
| Array Aggregator | 여러 번 실행된 결과를 하나의 배열로 모으기, 그룹화, 정렬, 중복 제거 |
| Text Aggregator | 여러 결과를 구분자·템플릿을 적용한 하나의 텍스트로 모으기 |
| Variables | 시나리오 실행 범위 변수 설정, 조회, 수정, 삭제 |
| Data Store | 사용자 소유 키·값 저장, 조회, 목록, 수정, 삭제, 만료시간 설정 |
| Error Handler | 재시도, 대체 경로, 무시, 실행 중단, 오류 데이터 전달 |

각 도구는 앱 모듈과 동일하게 작업 선택, 필수 입력 검증, 이전 단계 출력 매핑, 모듈 단독 테스트, 단계별 실행 기록을 지원한다. Data Store는 사용자별로 완전히 격리하며 Secret 저장소로 사용하지 않는다. Code와 수식 계산은 실행 시간, 출력 크기, 허용 API를 제한한다.

### 7.8 모든 API 작업 지원 원칙

수백 개 엔드포인트를 한 드롭다운에 무분별하게 노출하지 않는다. 각 앱은 자주 쓰는 공식 작업을 검색 가능한 기본 작업으로 제공하고, 기본 목록에 없는 작업은 `고급 > 사용자 지정 API 요청`으로 호출한다.

사용자 지정 API 요청은 선택한 앱의 저장된 인증을 서버에서 주입하며, 사용자는 Method, 공식 API URL, Query, Header, Body, 출력 선택만 설정한다. 이 조합으로 연결 계정의 권한 안에서 앱의 나머지 공식 API 엔드포인트도 실행할 수 있다.

## 8. 시나리오 저장 모델

각 노드는 최소한 다음 필드를 저장한다.

- `appId`
- `actionId`
- `credentialId`
- `position`
- `config`
- `inputMappings`
- `timeoutMs`
- `retryPolicy`
- `continueOnError`

앱 레지스트리 버전과 작업 스키마 버전을 함께 저장한다. 작업 정의가 변경되어 기존 노드가 호환되지 않으면 자동 실행을 막고 재설정을 요구한다.

실행 데이터는 사용자별로 격리해 저장한다.

- 실행: 시나리오, 버전, 시작 방식, 상태, 시작·종료 시각, 소요 시간, 오류 코드
- 단계 실행: 노드, 작업, 시도 횟수, 상태, 입력·출력 요약, 오류, 시작·종료 시각

## 9. 실행 흐름

1. 인증, 사용자 소유권, 결제 권한 또는 관리자 무료 권한 확인
2. 시나리오 그래프, 작업 정의, 필수 입력, 연결 상태 사전 검사
3. 활성 시나리오 버전을 실행 레코드로 고정
4. 그래프 순서대로 노드 실행
5. 이전 출력값을 다음 노드 입력으로 매핑
6. 단계 결과를 즉시 저장
7. 전체 실행 상태와 통계 갱신

기본 오류 정책은 다음과 같다.

- 429와 일시적 5xx는 지수 백오프로 제한된 횟수만 재시도
- 인증 만료와 권한 부족은 재시도하지 않고 재연결 안내
- 기본값은 실패 시 다음 노드 중단
- `실패해도 계속 실행`이 켜진 노드만 실패를 기록하고 다음 노드 실행
- 재실행은 새 실행 ID와 idempotency key를 생성
- 실행 중인 버전은 편집 중인 초안 변경의 영향을 받지 않음

## 10. 실제 실행과 준비 중 상태

앱 로고만 존재하고 실행기가 없는 가짜 연결 또는 가짜 성공 상태를 허용하지 않는다.

- 기본 실행기가 구현된 작업: 실제 API 호출 가능
- 사용자 지정 API 요청이 구성된 작업: 검증 후 실제 HTTP 호출 가능
- OAuth 운영자 설정이 없는 앱: `관리자 설정 필요`
- 연결 또는 작업 정의가 불완전한 모듈: `설정 필요`
- 아직 지원하지 않는 동작: `준비 중`으로 표시하고 실행 차단

현재 실행 API가 생성하는 모의 성공 결과는 실제 실행기와 단계별 실행 기록으로 교체한다.

## 11. 보안

- 모든 시나리오, 자격증명, 실행 기록 API는 로그인과 `ownerId`를 다시 검증한다.
- 구독하지 않은 사용자는 결제 권한 정책에 따라 자동화 API를 사용할 수 없다.
- 관리자 `kara111131@naver.com`은 기존 관리자 무료 사용 정책을 유지한다.
- API Key, Client Secret, Access Token, Refresh Token, Bot Token, Application Password는 AES-256-GCM으로 암호화한다.
- Client Secret과 저장된 비밀값은 브라우저로 반환하지 않는다.
- 비밀값은 로그, 오류, 입력·출력 미리보기에서 키 이름과 값 패턴을 함께 검사해 제거한다.
- OAuth state와 PKCE를 검증하고 Redirect URI를 정확히 고정한다.
- 사용자 지정 HTTP 요청은 localhost, loopback, link-local, 사설 IP, 클라우드 메타데이터 주소, DNS rebinding을 차단한다.
- 위험한 `Host`, `Cookie`, `Authorization`, 프록시 관련 헤더의 임의 덮어쓰기를 제한한다.
- URL은 HTTPS를 기본으로 하며 명시적으로 허용된 개발 localhost만 예외로 한다.
- 연결 검사는 응답 본문 전체를 저장하지 않고 성공 여부와 안전한 메타데이터만 저장한다.

## 12. 오류 UX

- 저장 실패는 로컬 캔버스 편집 상태를 유지하고 다시 저장할 수 있게 한다.
- 연결 오류는 잘못된 필드와 해결 방법을 앱별로 표시한다.
- 실행 실패 노드는 캔버스와 실행 상세에서 같은 색과 오류 코드로 표시한다.
- 권한 만료는 해당 앱의 연결 관리 화면으로 바로 이동하는 동작을 제공한다.
- 삭제 전에 연결을 사용하는 시나리오 목록과 개수를 표시한다.
- 빈 탭에는 설명과 단일 주요 동작을 제공한다.

## 13. 반응형과 접근성

- 제공된 데스크톱 시안의 3열 구성을 유지한다.
- 모든 그리드 자식에 `min-width: 0`을 적용해 글자가 영역 밖으로 삐져나오지 않게 한다.
- 긴 작업명과 연결명은 두 줄 제한과 전체 이름 툴팁을 사용한다.
- 작은 화면에서는 카탈로그와 설정 패널을 독립 서랍으로 전환한다.
- 캔버스만 pan·zoom하고 페이지 전체의 불필요한 가로 스크롤을 막는다.
- 키보드로 탭 전환, 모듈 추가, 작업 선택, 저장, 실행이 가능해야 한다.
- 아이콘 단독 버튼에는 접근 가능한 이름을 제공한다.

2. 자동화 화면 탭 분리와 실제 로고 적용
3. 모듈 작업 선택, 입력 매핑, 시나리오 저장 모델 확장

7. 사용자 지정 API 요청의 보안 실행기
8. 전체 테스트, 브라우저 검증, 커밋과 `main` 푸시

## 16. 완료 기준

- 기존 19개 모듈과 추가 20개 공통 앱이 실제 로고로 표시된다.
- 각 앱 연결 화면에 필요한 키·토큰·ID가 정확히 구분되어 표시된다.
- MAKE형 Text, 날짜·시간, Math, JSON, CSV, Aggregator, Variables, Data Store, Error Handler 도구가 작업 선택과 실제 실행을 지원한다.
- OAuth Client Secret은 일반 사용자가 입력하지 않는다.
- 모든 앱 모듈에서 지원 작업을 검색하고 선택할 수 있다.
- 기본 작업에 없는 공식 API는 사용자 지정 API 요청으로 실행할 수 있다.
- 템플릿, 실행 내역, 연결 관리, 사용 가이드 탭이 실제 데이터와 동작을 가진다.
- 연결되지 않거나 구현되지 않은 모듈은 실제 실행할 수 없고 가짜 성공을 반환하지 않는다.
- 실행 기록에는 단계별 실제 상태가 남고 비밀값은 남지 않는다.
- 사용자가 소유한 시나리오·자격증명·실행 기록만 접근할 수 있다.
- 글자와 로고가 컨테이너 밖으로 넘치지 않는다.
- 관련 자동화 테스트, 타입 검사, 빌드, 브라우저 검증이 통과한다.
- 구현 완료 후 변경사항을 `main`에 커밋하고 원격 저장소에 푸시한다.
